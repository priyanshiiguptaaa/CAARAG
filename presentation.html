Introduction
Context & Motivation

Large Language Models (LLMs) have achieved state-of-the-art results across NLP tasks. However, their knowledge is static and frozen at training time, making them vulnerable to outdated information and factual hallucinations.

Retrieval-Augmented Generation (RAG) was proposed to solve this — grounding LLM responses in live, external knowledge. Lewis et al. (2020) showed that RAG significantly reduces hallucination rates on knowledge-intensive benchmarks.

🔎 The Core Gap

Despite their success, all standard RAG systems share one critical flaw: they are static, single-pass pipelines with no ability to self-assess or self-correct when retrieval or generation fails.

⚠️

Miscalibration

LLMs are often overconfident even when their retrieved context is irrelevant or incomplete. Users cannot distinguish reliable from unreliable answers.

🔁

No Recovery Mechanism

If initial retrieval fails — e.g., missing a "bridge" document in multi-hop QA — the system has no fallback. It generates anyway, creating plausible hallucinations.

📉

No Trustworthiness Signal

There is no built-in way for users to know if an output from a RAG system is reliable or not, making production deployment risky.

03 / 12
Problem Statement
What Breaks in Standard RAG?
Problem 1

Retrieval Starvation

When the top-k documents don't contain the answer, the LLM is "starved" of evidence. Instead of failing, it fabricates a plausible response using parametric memory.

Problem 2

Answer Inconsistency

Multiple samplings of the same query produce wildly different answers, indicating high model uncertainty — but no existing pipeline detects or acts on this signal.

Problem 3

Uncalibrated Confidence

Even when models provide a confidence score, it is not calibrated (ECE > 0.3 in standard LLMs) — meaning stated confidence doesn't match actual accuracy.

🤔

The Core Question

"How can a RAG system know — and communicate — when it is uncertain, and automatically take corrective action?"

Expected Calibration Error (ECE) = Σ (|Bin| / N) × |Conf − Acc|

Standard RAG systems routinely achieve ECE > 0.25 — a critical reliability failure

Research Objectives
What We Aim to Achieve
01
Hybrid Retrieval Architecture

Design a combined dense (SBERT) + sparse (BM25) retrieval pipeline fused via Reciprocal Rank Fusion (RRF) to maximize document recall.

02
Triple-Signal Confidence (TSC)

Formalize three diagnostic signals — Retrieval Stability (Sᵣ), LLM Self-Confidence (Sₗ), and Semantic Consistency (Sᶜ) — to quantify multi-dimensional uncertainty.

03
Adaptive Diagnostic Controller

Implement a signal-aware loop that identifies specific failure modes and triggers targeted remedial actions (expansion or reformulation) — rather than blindly retrying.

04
Calibration & Empirical Validation

Achieve an Expected Calibration Error (ECE) < 0.10 and demonstrate that the confidence score is a reliable predictor of answer quality (Pearson r ≥ 0.5) on HotpotQA.

Methodology
The 5-Stage Diagnostic Adaptive Pipeline
📖

Stage 1: Corpus Preparation

Wikipedia chunking (150-word windows), text normalization, BM25 + FAISS dual indexing via SBERT (all-mpnet-base-v2)

↓
🔍

Stage 2: Hybrid Retrieval (RRF)

Dense (SBERT cosine) + Sparse (BM25) retrieval fused via Reciprocal Rank Fusion → top-k context chunks

↓
🤖

Stage 3: Multi-Sample Generation

Groq LLM (Llama-3.3-70B) generates N=5 stochastic samples. Each outputs: Answer + Explanation + Self-Confidence

↓
📊

Stage 4: Signal Computation & Calibration

Compute Sᵣ, Sₗ, Sᶜ → weighted z-score → sigmoid calibration C ∈ [0,1]

↓
🔄

Stage 5: Adaptive Controller (Diagnostic Loop)

If C < τ=0.7: Diagnose weakest signal → apply k-expansion or query reformulation → repeat (max 3 rounds)

RRF(d) = Σ 1 / (β + Rank_m(d))
β=60, m ∈ {Dense, Sparse}
z = w₁·Sᵣ + w₂·Sₗ + w₃·Sᶜ
C = 1 / (1 + e^−(a·z + b))
Sigmoid Platt Scaling calibration
🔑 Key Innovation

Unlike AIR-RAG (generic retry), our system diagnoses which of the 3 signals is the bottleneck and applies the specific corrective action — reducing wasted retrieval rounds by up to 40%.

Confidence Modeling
Triple-Signal Confidence (TSC) Framework
📡
Sᵣ — Retrieval
Normalized average RRF score of top-k docs. Low Sᵣ = sparse evidence coverage → triggers k-expansion
🧠
Sₗ — LLM
Average self-reported confidence from structured prompt output across N=5 samples. Low Sₗ = model uncertain about context
🔀
Sᶜ — Consistency
Pairwise SBERT cosine similarity across N answer embeddings. Low Sᶜ = conflicting answers → query reformulation

Diagnostic Decision Matrix

OKOKLowInconsistentReformulate

HighHighHigh✅ ConfidentOutput answer
Sᶜ = (1/N²) ΣᵢΣⱼ CosineSim(Embed(Ansᵢ), Embed(Ansⱼ))
🔑 Calibration Layer

The sigmoid parameters (a, b) are learned on a held-out validation set via Platt Scaling — ensuring C is a true probability, not a heuristic score.

Adaptive Threshold: τ = 0.7

If C ≥ 0.7, the system outputs the answer. If C < 0.7 after 3 rounds, the system gracefully reports low confidence — preventing hallucination.

Discussion
Key Insights & Findings
🎯

Confidence Predicts Quality

With r=0.99 Pearson correlation, our calibrated C is a near-perfect predictor of answer quality. The system demonstrably "knows when it doesn't know" — a critical first step for production reliability.

🔄

Retrieval Starvation is the Main Cause

80%+ of failures are due to incomplete initial retrieval (low Sᵣ), not LLM generation failures. The k-expansion adaptation resolves the majority of these cases within one additional round.

📐

Calibration Works

ECE = 0.064 is in the "Excellent" range (< 0.10). The Sigmoid Platt Scaling approach removes the inherent overconfidence bias seen in standard LLMs (ECE typically > 0.25).

🆚

Diagnosis vs. Blind Retry

Unlike AIR-RAG which blindly retries, our diagnostic loop reduces redundant rounds by 40% by targeting the exact failure dimension — a more compute-efficient adaptation strategy.

⚠️

Known Limitation: Corpus Bounds

If the answer is genuinely absent from the corpus, no adaptation will help. However, the system correctly identifies these cases and reports low confidence — gracefully refusing to hallucinate.

Key Takeaway: The framework proves that trustworthy RAG requires not just better retrieval — but self-awareness. A model that knows its limits is safer than one that doesn't.
